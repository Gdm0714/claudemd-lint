# Project Instructions

## Build & Run

- Use `npm run dev` to start the development server
- Always run `npm test` before committing
- Use npm run dev to start the dev server

## Code Style

- Always use TypeScript strict mode
- Never use `any` type in TypeScript code
- Use single quotes for strings
- Always prefer single quotes over double quotes for strings
- Never use var, always use const or let

## Testing

- Run tests with `npm test`
- Always write unit tests for new functions
- Use vitest for testing
- Make sure to always write tests before submitting PRs

## Conventions

- Use kebab-case for file names
- Always use kebab-case for filenames
- Never use camelCase for file names
- Follow the conventional commits spec for commit messages

## Deployment

- Always run the linter before deploying
- Use `npm run build` before deployment
- Never deploy on Fridays
- Always deploy to staging first

## Git

- Use conventional commits
- Never force push to main branch
- Always create feature branches from develop
